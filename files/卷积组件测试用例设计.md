# Convolution1d / Convolution2d 组件测试用例设计

> 文档目标：从黑盒 + 边界值 + 等价类 + 特殊值的测试视角，为 behavior 工具中的卷积组件
> `Convolution1d`、`Convolution2d` 设计系统化、可复现、覆盖各类边界场景的测试用例。

---

## 1. 被测对象与接口分析

### 1.1 组件接口

| 项 | 说明 |
|---|---|
| 输入端口 `signal` | 待卷积数据（1D：向量 / 2D：矩阵） |
| 输入端口 `kernel` | 卷积核（1D：向量 / 2D：矩阵） |
| 输入类型（可配置） | `uint8` / `int8` / `uint16` / `int16` / `uint32` / `int32` / `float` / `double`（共 8 种，signal 与 kernel 可独立配置） |
| 输出类型（界面设置） | `double` / `float`（仅 2 种） |
| 形状 `shape`（界面设置） | `valid` / `same` / `full`（共 3 种） |

### 1.2 可变测试因子（参数维度）

| 因子 | 取值数 | 取值 |
|---|---|---|
| 组件 | 2 | Convolution1d / Convolution2d |
| signal 类型 | 8 | uint8/int8/uint16/int16/uint32/int32/float/double |
| kernel 类型 | 8 | 同上 |
| 输出类型 | 2 | double / float |
| shape | 3 | valid / same / full |
| signal 尺寸 | 多 | 见尺寸边界 |
| kernel 尺寸 | 多 | 见尺寸边界 |
| 数据取值 | 多 | 普通值 / 类型边界常量 / 特殊值(Inf/NaN) |

> 全组合规模巨大（仅类型×输出×shape = 8×8×2×3 = 384，再乘尺寸与取值上万级）。
> 因此采用 **等价类划分 + 边界值分析 + 成对(pairwise)组合** 缩减用例，详见第 5 章。

## 2. 卷积运算定义与测试 Oracle

测试预期值（Oracle）由数学定义独立计算（建议用 NumPy / SciPy 的 `numpy.convolve`、`scipy.signal.convolve2d` 双重比对，注意工具语义是否为"卷积"而非"相关"）。

### 2.1 卷积 vs 相关（必须验证核翻转）

- **卷积（convolution）**：核会被翻转（1D 反转、2D 旋转 180°）。
- **相关（correlation）**：核不翻转。

> ⚠ 这是最易出错点：用**对称核无法区分两者**，因此核翻转类用例必须使用**非对称核**。

### 2.2 一维卷积

signal 长度 `N`，kernel 长度 `K`：

```
full(x, h)[n] = Σ_k x[k] · h[n-k]
```

| shape | 输出长度 | 含义 |
|---|---|---|
| full | `N + K - 1` | 任意重叠（含边缘补零） |
| same | `N` | 与 signal 等长，取 full 的居中部分 |
| valid | `N - K + 1`（要求 N≥K） | 仅 signal 与 kernel 完全重叠的位置（无补零） |

**Oracle 示例（核翻转验证用例，非对称核）**
- `x = [1, 2, 3]`，`h = [0, 1, 2]`
- `full  = [0, 1, 4, 7, 6]`（长度 5 = 3+3-1）
- `same  = [1, 4, 7]`（长度 3，取居中 3 个）
- `valid = [4]`（长度 1 = 3-3+1）

> 若组件错误实现为"相关"，上述 full 会变为 `[2, 5, 8, 3, 0]`，可立即区分。

### 2.3 二维卷积

signal `M×N`，kernel `P×Q`：

```
full(x, h)[i,j] = Σ_m Σ_n x[m,n] · h[i-m, j-n]   (核旋转 180°)
```

| shape | 输出尺寸 |
|---|---|
| full | `(M+P-1) × (N+Q-1)` |
| same | `M × N`（居中裁剪） |
| valid | `(M-P+1) × (N-Q+1)`（要求 M≥P 且 N≥Q） |

**Oracle 示例**
- `x = [[1,2],[3,4]]`，`h = [[1,2],[3,4]]`
- `full = [[1,4,4],[6,20,16],[9,24,16]]`（3×3）
- `valid = [[20,16],[24,16]]`（2×2）
- `same = [[1,4],[6,20]]`（2×2，居中裁剪，偶数核偏移以实现为准）

## 3. 类型边界常量映射表

按用户提供常量整理每种输入类型的取值边界，用于"数据值边界"用例填充。

| 类型 | 最小值 | 最大值 | 备注 |
|---|---|---|---|
| uint8 | `0ub` | `255ub` | 8-bit 无符号 |
| int8 | `-128` | `127` | 8-bit 有符号 |
| uint16 | `0` | `65535` | 16-bit 无符号 |
| int16 | `-32768s`(MinShort) | `32767s`(MaxShort) | 16-bit 有符号 |
| uint32 | `0` | `4294967295` | 32-bit 无符号 |
| int32 | `-2147483648`(MinInt) | `2147483647`(MaxInt) | 32-bit 有符号 |
| int64(long)* | `-9223372036854775808L` | `9223372036854775807L` | 若支持 |
| float | `±1.401298464324817E-45`(MinFloat,denorm) | `±3.402823466385289E38`(MaxFloat) | 上溢→`Infinityf`(1.0f/0.0f) |
| double | `±4.9E-324`(MinDouble,denorm) | `±1.797693134862316E308`(MaxDouble) | 上溢→`Infinity`(1.0/0.0) |

其它关键常量：
- `eps = 2.220446049250313E-16`（double 机器精度，用于精度断言阈值）
- `e = 2.718281828459`，`pi = 3.1415926535898`（一般浮点取值）
- 浮点精度无损整数范围：float ≤ `2^24 = 16777216`，double ≤ `2^53 ≈ 9.007E15`（超出即丢精）

## 4. 测试分类总览

| 类别 | 编号前缀 | 关注点 |
|---|---|---|
| A 功能正确性 | `-F-` | 卷积结果、核翻转、shape 尺寸、输出类型 |
| B 尺寸边界 | `-S-` | 核/信号尺寸的极端组合、奇偶、空、超长核 |
| C 数据值边界 | `-V-` | 各类型 Max/Min/0/1、整数溢出、浮点上溢、精度 |
| D 特殊值 | `-X-` | Inf / -Inf / NaN 传播（仅 float/double） |
| E 异常鲁棒性 | `-E-` | 空输入、维度不匹配、valid 下核>信号 |
| F 精度/性能 | `-P-` | float vs double 精度差、大规模数据 |

用例 ID 规则：`CV1`（Convolution1d）/ `CV2`（Convolution2d）+ 类别 + 序号。
通用断言：① 输出尺寸正确；② 数值与 Oracle 在容差内一致；③ 输出数据类型符合设置；④ 无崩溃/异常码符合预期。

---

## 5. 组合缩减策略

1. **核翻转、shape、尺寸边界**：与类型无关的逻辑缺陷 → 固定用 `double` 类型穷尽这些维度。
2. **类型组合**：用 pairwise 覆盖 signal类型 × kernel类型 × 输出类型 的成对组合，而非全组合。
3. **数据值边界**：对每种类型单独构造 Max/Min 用例（不与所有 shape 交叉，固定 shape=`full`）。
4. **特殊值**：仅对 float/double 设计。
5. 关键风险组合（如 uint32 大值 + float 输出 → 丢精/上溢）单独强化。

## 8. 输出类型（double / float）专项

输出类型与输入类型的交叉是高风险区，单列强化：

| 场景 | 输入 | 输出 | 预期重点 |
|---|---|---|---|
| 大整数丢精 | uint32/int32 满值 | float | 相对误差显著（float 仅 24 位尾数），断言"丢精但不崩溃" |
| 中等整数丢精 | uint16 满值乘加 | float | 超过 2^24 部分丢精 |
| double 上溢 float | double 值在 (MaxFloat, MaxDouble) 区间 | float | 结果转为 +Infinity / -Infinity |
| 精度保持 | 小整数/小浮点 | float & double | 两者一致或在 eps/float-eps 容差内 |
| 负方向上溢 | 负的极大值 | float/double | -Infinity |
| 次正规数 | MinFloat/MinDouble | 对应类型 | 不被错误截断为 0 |

---

## 9. 特殊值传播规则参考（断言依据）

| 运算 | 结果 |
|---|---|
| 正数 × (+Inf) | +Inf |
| 负数 × (+Inf) | -Inf |
| 0 × (±Inf) | NaN |
| (+Inf) + (-Inf) | NaN |
| 任意 op NaN | NaN |
| ±Inf 落入 float 范围 | 保持 ±Inf |
| double 值 > MaxFloat 转 float | ±Inf |

> 整数类型（uint8…int32）**无法表示 Inf/NaN**，故 D 类用例只对 float/double 设计；
> 但整数输入 + 浮点输出仍可能因上溢/丢精产生大数或 ±Inf，归入 C/第 8 章。

---

## 10. 组合覆盖矩阵（pairwise 建议）

类型组合不做全 64 组，按成对覆盖抽样（示例，shape 与输出类型轮转分布）：

| # | signal 类型 | kernel 类型 | 输出 | shape |
|---|---|---|---|---|
| 1 | uint8 | uint8 | double | full |
| 2 | int8 | uint16 | float | same |
| 3 | uint16 | int16 | double | valid |
| 4 | int16 | uint32 | float | full |
| 5 | uint32 | int32 | double | same |
| 6 | int32 | float | float | valid |
| 7 | float | double | double | full |
| 8 | double | float | float | same |
| 9 | float | float | double | valid |
| 10 | double | double | float | full |
| 11 | uint8 | int32 | float | valid |
| 12 | int8 | double | double | same |

> 目标：每个"输入类型"至少作为 signal、kernel 各出现一次；输出类型与 shape 在矩阵中均衡覆盖；
> 任意 (signal类型, kernel类型)、(类型, 输出)、(类型, shape) 二元对尽量被覆盖。

## 11. 测试执行与断言约定

1. **Oracle 来源**：NumPy `numpy.convolve` / SciPy `scipy.signal.convolve2d`，`mode` 与 shape 对应（full/same/valid）。必须先确认组件语义为"卷积"（核翻转）。
2. **数值容差**：
   - double：绝对/相对误差 ≤ `1e-9`（普通值）；大值用相对误差 ≤ `1e-12 × 量级`。
   - float：相对误差 ≤ `1e-5`。
   - 整数精确路径（结果在无损范围内）：要求**完全相等**。
3. **特殊值断言**：用 `isnan` / `isinf` / 符号判定，不能用普通相等比较。
4. **尺寸断言**：先比对输出形状，再比对数值。
5. **异常用例**：断言抛出预期错误/返回约定的空结果，且进程不崩溃。

## 12. 覆盖度小结

| 维度 | 覆盖方式 |
|---|---|
| shape（valid/same/full） | A 类穷尽 + 尺寸边界各 shape 交叉 |
| 输入类型（8 种） | C 类逐类型边界 + 第 10 章 pairwise |
| 输出类型（double/float） | 第 8 章专项 + pairwise 轮转 |
| 核翻转正确性 | CV1-F-004 / CV2-F-004（非对称核） |
| 尺寸边界 | B 类（1×1、K=N、K>N、空、奇偶、大规模、非方阵、行/列向核） |
| 数值边界 | C 类（各类型 Max/Min/0/次正规、整数溢出、浮点上溢、eps） |
| 特殊值 | D 类（Inf/-Inf/NaN 传播） |
| 异常鲁棒性 | E 类 |
| 精度/性能 | F 类 + 第 8 章 |

> 用例总量（代表性）：Convolution1d ≈ 50+，Convolution2d ≈ 45+，可按 pairwise 矩阵进一步实例化为脚本化参数用例。
